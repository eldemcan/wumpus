Wumpus Game.

Hunt The Wumpus game project. It is a classic assignment given for students who works on AI. Hunt the Wumpus is an early video game, based on a simple hide and seek format featuring a mysterious monster (the Wumpus) that lurks deep inside a network of rooms.(wiki)

You can watch the game from here.

Rules of game

The world contains a Wumpus, a pile of gold,bat and any number of pits. Each of these items is in a square of the grid. No square contains two of these items, some squares contain none of these items, and the items do not move. Goal of the robot is to get to the square with the gold and reach the exit, without running into the Wumpus or a pit. The robot has one arrow with which it can shoot and kill the Wumpus. If agent encounters with a bat, bat teleport agent into a random block.

Game Board

Game board has the topology of a Mobius Strip shape which means one edge of the board connected with other edge.

Game Play

Agent should move from square to square, i.e. at all times the robot should be either stopped completely in a square, or should be crossing from one square into an adjacent square to the left, right, up, or down from it. There will be physical smells, breezes, or glitter.

As agent explore it will encounter 3 types of rooms:

Normal Rooms - These rooms are empty. Cold Rooms - These indicate a pit is nearby. Smelly Rooms - These indicate the Wumpus is nearby.

You can assume that the world is such that an infinitely smart being can solve the puzzle without guessing.(depending on the perceptions)

Deployment

Project run environment

windows 7 - javaSE 1.6 ubuntu 12 - javaSE 1.6

  • import the project into eclipse
  • setup class includes settings about game like table size bat size please be careful when you set numbers e.g don't put more pit than cave size etc.
  • game class is main class of project if you like to play game by yourself switch off AI by changing ai_switch=true variable with false otherwise AI will play game itself and show output on screen.
